## 1. 组织结构

朝鲜的网络攻击并非个人或民间团体的行为，而是直属于**朝鲜劳动党侦察总局（RGB, Reconnaissance General Bureau）**、由国家主导的系统性网络战力。该机构于2009年整合劳动党作战部、第35室及人民武力部侦察局而成立，由六个局组成，直接指挥核心黑客部队。

> ⚠️ **国家级黑客组织的意义**  
> 这些并非普通犯罪团伙，而是接受国家预算和军事技术支持的网络战力。联合国安理会报告多次确认，通过黑客手段盗取的资金被转用于核武器和弹道导弹的开发。

### 主要黑客部队

| 组织名称 | 别名 | 主要任务 | 代表性攻击 |
|---|---|---|---|
| **拉撒路（Lazarus）** | Zinc / Hidden Cobra | 金融盗窃、基础设施破坏、全面APT | 索尼影业、Upbit、Bybit |
| **蓝诺罗夫（Bluenoroff）** | 拉撒路下属 | 专攻金融机构与加密货币交易所 | 孟加拉国央行、SWIFT攻击 |
| **安达利尔（Andariel）** | Silent Chollima | 勒索软件、凭证窃取、国防黑客 | Upbit共谋、10余家防卫企业 |
| **金秀基（Kimsuky）** | Thallium | 情报收集、鱼叉式钓鱼、外交安全机密 | 韩水原黑客事件、外交渠道渗透 |
| **斯卡克劳夫特（Scarcruft）** | APT37 | 渗透韩国、日本、中东民间/公共部门 | 太永浩智能手机黑客事件 |

> 📊 安博实验室2025年报告：朝鲜APT攻击86起，远超中国（27起）和俄罗斯（18起）。

## 2. 黑客历史

朝鲜的黑客战略逐步演进——早期通过销售游戏外挂程序赚取外汇，随后渗透金融网络、发动全球勒索软件攻击，最终锁定加密货币盗窃。

### 📅 时间轴

**2000年代初——网络游戏外挂程序（首个外汇创收手段）**  
通过朝鲜龙鹤岛贸易总公司和朝鲜电脑中心，开发并销售《天堂》、《枫之谷》、《地下城与勇士》等游戏的外挂程序。收益经第39号室流入统治资金。一名私服运营者因购买朝鲜黑客的入侵程序并向中国银行账户汇款2,380万韩元，被以违反《国家安全法》罪名逮捕。

**2009年——特洛伊行动（Operation Troy）**  
以韩国军队和驻韩美军为目标，恶意软件在政府和军事计算机中潜伏至少四年，收集军事机密，同期对青瓦台网站发动DDoS攻击。

**2014年——索尼影业黑客事件 / 韩水原黑客事件**  
为报复电影《采访》（讲述暗杀金正恩），入侵并摧毁索尼内部网络。FBI正式指认朝鲜为幕后黑手。国内方面，韩国水力原子力株式会社的核电站图纸遭窃。

**2016年——孟加拉国央行黑客事件 `盗取8,100万美元`**  
黑入SWIFT系统，向美联储发出虚假汇款指令。通过菲律宾赌场洗钱，系蓝诺罗夫所为。

**2017年——WannaCry勒索软件 `[严重]`**  
感染150个国家的30万台计算机，英国NHS医疗系统瘫痪。利用美国国家安全局泄露的EternalBlue漏洞。美英政府正式谴责朝鲜。

**2019年——Upbit以太坊盗币案 `盗取580亿韩元`**  
盗取342,000枚ETH。经五年调查确认为拉撒路与安达利尔合谋。按当前市价约合14,700亿韩元。

**2024年——全面进攻国防企业**  
拉撒路、安达利尔、金秀基三个组织同时出动，攻击83家国防企业，成功入侵10余家。

**2025年——Bybit黑客事件 `盗取14.6亿美元`**  
仅凭一台开发者电脑感染，完成史上最大规模单笔加密货币盗窃。2025年朝鲜盗取总额突破20亿美元。

## 3. Upbit盗币事件

2019年11月的Upbit黑客事件标志着朝鲜战略的决定性转变——从传统金融体系全面转向**加密货币交易所**。

```
[事件概要]
盗取规模 : 342,000枚ETH（当时约5,800万美元 / 当前约1.47万亿韩元）
发生时间 : 2019年11月
官方确认 : 2024年11月（韩国警察厅国家搜查总部）
幕后组织 : 拉撒路（Lazarus）+ 安达利尔（Andariel）合谋
调查协作 : FBI协助，确认朝鲜IP地址及朝鲜语词汇（'헐한 일'）
```

### 盗币后洗钱路线

1. **朝鲜自建交易所（57%）**——通过朝鲜自主设计运营的3个匿名交易所，以低于市价2.5%的价格将ETH换成BTC，无需外部KYC即可立即变现。
2. **经13个国家51家交易所洗钱（43%）**——通过中国、美国、香港、瑞士等全球交易所分散流转，混淆资金来源，使国际追踪复杂化。
3. **中国场外经纪商与地下金融网络**——通过与银联卡关联的账户将加密货币转换为法定货币，与三合会等犯罪组织合作完成最终变现。

## 4. Telegram渗透手法

Telegram的加密通信、匿名性和机器人自动化功能使其成为加密货币社区的主要沟通渠道，也成为朝鲜传播恶意软件、接触受害者的核心途径。

| 手法 | 内容 |
|---|---|
| **伪造钱包APP传播** | 复制MyEthereum界面的"Somora"APP通过Telegram链接传播，绕过应用商店安全审查。 |
| **伪装投资频道** | 通过高收益投资提案频道/私信建立信任后，引导安装"专用投资平台"（恶意APP）。 |
| **冒充Zoom会议** | 冒充熟人发送Telegram消息 → Calendly预约 → 发送虚假Zoom邀请 → 诱导执行"Zoom SDK更新脚本"。恶意代码隐藏在一万行空白之后。 |
| **AI深度伪造身份** | 利用AI生成的虚假面孔和证件冒充投资者或公众人物，视频通话中亦难以识别。 |

### 恶意软件功能清单（NimDoor系列）

```
键盘记录器   : 记录全部键盘输入 → 窃取密码和助记词
剪贴板监控   : 实时将复制的钱包地址替换为攻击者地址
浏览器数据   : 窃取存储的凭证、Cookie和会话信息
钱包扩展程序 : 窃取包括MetaMask在内的24个以上钱包扩展数据
远程控制     : 远程操控摄像头和麦克风，传输或删除文件
Telegram数据 : 运行脚本（tlgrm）窃取Telegram本地数据
持久化机制   : 注册LaunchAgent（macOS），重启后自动运行
```

## 5. 开发者伪装渗透

2024至2025年最突出的战术是**伪装成IT开发者或工程师，正式入职加密货币企业**。据Chainalysis报告，仅2024年一年就有12家以上的加密企业被朝鲜特工以合法员工身份渗透。

> ⚠️ **边界安全的瓦解**  
> 当攻击者已拥有合法的内部访问权限时，防火墙和网络边界安全将形同虚设。这一战略在整个行业引发了根本性的信任危机。

### 伪装入职攻击步骤

1. **构建虚假身份**——结合AI生成的照片、虚构履历和幽灵公司背景，在LinkedIn和GitHub上积累一段时间的活动记录。
2. **申请远程职位**——应聘区块链开发者、智能合约工程师等高需求岗位，具备通过技术面试的实际能力。
3. **内部侦察**——以合法员工身份工作，悄悄摸清内部基础设施、安全架构和私钥管理方式，历时数月。
4. **执行——从内部协助攻击**——提供凭证、禁用安全系统、批准欺诈交易，直接从内部协助外部黑客团队。

| 手法 | 内容 |
|---|---|
| **蚂蚁群式洗钱（Smurfing）** | 将资金拆分为50万美元以下，通过数千次分散转账瓦解国际监控网络。 |

| **经币安洗钱** | ICIJ调查记者报道：约9亿美元朝鲜黑客收益流入5个币安账户，有交易记录为证。 |

## 9. 防御清单

> 💡 **核心原则**：朝鲜的攻击针对的是人类的信任与习惯，而非技术漏洞。技术层面的安全加固必须辅以以人、流程和供应链为核心的安全文化。

### 基本防御措施

- [ ] **Telegram私信**：绝对不回应陌生人的投资邀约、文件或链接。若疑似冒充熟人，须通过其他渠道核实身份。
- [ ] **硬件钱包**：将大额资产存入冷钱包，热钱包仅保留日常交易所需小额资金。
- [ ] **剪贴板劫持**：发送前必须肉眼核对完整钱包地址，并使用剪贴板监控安全工具。
- [ ] **工作邀约邮件**：立即拒绝运行任务文件的要求，并通过官方渠道验证身份。
- [ ] **开源代码**：执行前手动检查`.vscode/tasks.json`和`package.json`脚本，禁用VSCode自动运行。
- [ ] **供应链安全**：定期审计第三方库和服务；聘用外包开发者时进行背景调查。

### 双因素认证（2FA）——账户被盗的最后防线

> 🔐 **没有2FA的账户就是没有锁的保险箱**  
> 即使密码被盗，启用了2FA的账户也让攻击者无法通过第二重认证。

| 方式 | 安全等级 | 说明 |
|---|---|---|
| **硬件安全密钥（FIDO2/U2F）** | 🟢 最强 | 如YubiKey等物理密钥，在钓鱼网站输入时域名不匹配则认证自动失败。强烈推荐用于交易所和高价值账户。 |
| **TOTP应用（Google身份验证器 / Authy）** | 🟡 推荐 | 每30秒刷新一次的6位OTP，比短信更安全且可离线使用。备份码必须离线保存。 |
| **短信验证码（SMS 2FA）** | 🟠 不推荐 | 易受SIM卡调换攻击和SS7协议漏洞的影响，须立即换用TOTP或硬件密钥。 |
| **邮件验证码（Email 2FA）** | 🔴 禁止使用 | 电子邮件账户本身就是钓鱼的主要目标，邮件一旦被攻破，此2FA立即失效。 |

**2FA优先级**：交易所账户 > 邮件账户 > Telegram/社交媒体 > 云存储  
使用交易所API密钥时，须将权限限制为**禁止提现/只读**，并同步设置IP白名单。

### 网吧与公共网络——严禁访问加密货币

> ⛔ **使用公共电脑的四大威胁**
> 1. **键盘记录器无法被发现**——恶意软件或键盘记录器可能已被安装，肉眼无法辨别。
> 2. **浏览器会话残留**——即使退出登录，认证信息仍可能残留在缓存和Cookie中，暴露给下一位用户。
> 3. **公共Wi-Fi中间人攻击（MITM）**——在开放网络（咖啡馆、图书馆、机场）中，同一网络上的攻击者可拦截流量或将用户引导至伪造热点。
> 4. **旁窥攻击（Shoulder Surfing）**——在公共场所输入助记词和密码，可能被周围人或监控摄像头看到。

**✅ 必须遵守的规则：**
- [ ] 仅使用本人专属设备（智能手机或笔记本电脑）访问加密货币交易所和钱包
- [ ] 若不得不使用公共电脑，绝对不输入助记词、私钥或密码
- [ ] 使用公共Wi-Fi时必须先连接可信的付费VPN
- [ ] 仅从官方应用商店安装交易所应用，严禁通过二维码或链接安装
- [ ] 屏幕共享或直播时绝对不暴露2FA验证码（直播者/YouTuber须格外注意）

**❌ 严格禁止：**
- 使用网吧后仅清除浏览器记录（若已感染键盘记录器，此举毫无意义）
- 未使用个人热点，在网吧有线网络上访问个人钱包或交易所
- 在他人设备上登录交易所（"只看一眼"）

### 交易所及项目运营方专属清单

**✅ 必须执行：**
- [ ] 热钱包资产占比限制在总量的10%以下，其余存入冷存储
- [ ] 对多签钱包的签名基础设施本身进行安全审计，包括第三方服务（Bybit的教训）
- [ ] 所有管理员账户强制使用基于FIDO2硬件密钥的2FA
- [ ] 远程招聘时，考虑深度伪造风险，引入额外的身份核实程序
- [ ] 实时监控内部人员权限访问日志，设置异常行为即时告警
- [ ] 为开发团队电脑部署端点安全解决方案，限制安装外部软件

**❌ 严格禁止：**
- 未经验证将来自不可信来源的SDK或库直接部署到生产环境
- 大额提现请求仅由自动化系统处理（无人工核验环节）
- 未实时比对朝鲜制裁区块链地址数据库即批准交易
