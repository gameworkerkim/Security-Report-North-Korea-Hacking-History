## 1. 組織構造

北朝鮮のサイバー攻撃は、個人や民間団体の活動ではない。**朝鮮労働党偵察総局（RGB, Reconnaissance General Bureau）**直属で運営される国家レベルの体系的なサイバー戦力である。2009年に労働党作戦部、第35室、人民武力部偵察局を統合して設立されたこの機関は6つの局で構成され、核心ハッキング部隊を直接指揮している。

> ⚠️ **国家支援型ハッキング組織の意味**  
> 単なる犯罪集団ではなく、国家予算と軍事技術支援を受けている。ハッキングで盗取した資金が核兵器・弾道ミサイル開発の財源に転用されていることは、国連安全保障理事会の報告書によって繰り返し確認されている。

### 主要ハッキング部隊

| 組織名 | 別名 | 主要任務 | 代表的な攻撃 |
|---|---|---|---|
| **ラザルス（Lazarus）** | Zinc / Hidden Cobra | 金融盗取、インフラ破壊、広範囲APT | ソニー・ピクチャーズ、Upbit、Bybit |
| **ブルーノロフ（Bluenoroff）** | ラザルス傘下 | 金融機関・仮想通貨取引所専門の金銭盗取 | バングラデシュ中央銀行、SWIFT攻撃 |
| **アンダリエル（Andariel）** | Silent Chollima | ランサムウェア、認証情報窃取、防衛産業ハッキング | Upbit共謀、防衛企業10社超への侵入 |
| **キムスキー（Kimsuky）** | Thallium | 情報収集、スピアフィッシング、外交・安全保障機密 | 韓国水力原子力ハッキング、外交ルート侵入 |
| **スカークラフト（Scarcruft）** | APT37 | 韓国・日本・中東の民間/公共部門への侵入 | 太永浩スマートフォンハッキング |

## 2. ハッキングの歴史

北朝鮮のハッキング戦略は段階的に進化した。初期はオンラインゲームのBOTプログラム販売で外貨を稼ぎ、その後、金融ネットワーク侵入、グローバルランサムウェア攻撃、仮想通貨盗取へとターゲットと手法が精緻化していった。

### 📅 タイムライン

**2000年代初頭——オンラインゲームBOTプログラム（最初の外貨獲得手段）**  
朝鮮龍鶴島貿易総会社・朝鮮コンピュータセンターを通じ、リネージュ・メイプルストーリー・ダンジョン&ファイター向けのBOTプログラムを製作・販売。収益は39号室を経由して統治資金へ流入。あるプライベートサーバー運営者が北朝鮮ハッカーからハッキングプログラムを購入し、中国の銀行口座へ2,380万ウォンを送金した結果、国家保安法違反で逮捕された。

**2009年——トロイ作戦（Operation Troy）**  
韓国軍および在韓米軍を標的に、マルウェアが少なくとも4年間にわたって政府・軍のコンピュータに潜伏して軍事機密を収集。同時に青瓦台のウェブサイトへDDoS攻撃を実施。

**2014年——ソニー・ピクチャーズハッキング / 韓国水力原子力ハッキング**  
映画『ザ・インタビュー』（金正恩暗殺を描いた作品）への報復としてソニーの内部ネットワークを破壊。FBIが北朝鮮を正式に名指し。国内では韓国水力原子力の原発図面が盗取された。

**2016年——バングラデシュ中央銀行ハッキング `8,100万ドル盗取`**  
SWIFTシステムをハッキングし、米国連邦準備銀行へ虚偽の送金指示を送信。フィリピンのカジノでマネーロンダリング。ブルーノロフの犯行。

**2017年——WannaCryランサムウェア `[重大]`**  
150カ国の30万台のコンピュータに感染。英国NHSが麻痺。NSAから流出したEternalBlue脆弱性を悪用。米英政府が北朝鮮を正式に非難。

**2019年——Upbitイーサリアム盗取事件 `580億ウォン盗取`**  
ETH34万2,000個を盗取。5年間の捜査を経て、ラザルスとアンダリエルの共謀が確認された。現在の時価に換算すると約1兆4,700億ウォン相当。

**2024年——防衛産業企業への総力戦**  
ラザルス・アンダリエル・キムスキー3組織が同時投入。83の防衛企業を攻撃し、10社超への侵入に成功。

**2025年——Bybitハッキング `14.6億ドル盗取`**  
開発者PC1台の感染から史上最大規模の単発仮想通貨盗取事件が実現。2025年の北朝鮮の盗取総額は20億ドルを突破。

## 3. Upbitハッキング

2019年11月のUpbitハッキングは、北朝鮮の戦略が伝統的な金融から**仮想通貨取引所**へ本格的に転換した分岐点である。

```
[事件概要]
被害規模 : ETH 342,000個（当時約5,800万ドル / 現在約1.47兆ウォン相当）
発生日時 : 2019年11月
公式確認 : 2024年11月（韓国警察庁国家捜査本部）
背後組織 : ラザルス（Lazarus）+ アンダリエル（Andariel）の共謀
捜査協力 : FBI協力、北朝鮮IPアドレス・朝鮮語語彙（'헐한 일'）確認
```

### 盗取後のマネーロンダリング経路

1. **北朝鮮独自の取引所を活用（57%）**——北朝鮮が設計・運営する匿名取引所3カ所で、ETHを市価より2.5%低い価格でBTCに交換。外部KYCなしで即座に現金化。
2. **13カ国51取引所を経由してマネーロンダリング（43%）**——中国・米国・香港・スイスなど世界中の取引所を分散活用し、資金の出所を混合することで国際的な追跡を複雑化。
3. **中国OTCブローカーと地下金融ネットワーク**——銀聯カードに連動した口座を通じて法定通貨へ変換。三合会などの犯罪組織と協力して最終的に現金化。

## 4. Telegram侵入手法

Telegramは暗号化通信・匿名性・ボット自動化の構造により仮想通貨コミュニティの主要な通信チャンネルとなっており、北朝鮮はこれをマルウェア配布と被害者へのアクセスの中核チャンネルとして利用している。

| 手法 | 内容 |
|---|---|
| **偽ウォレットアプリの配布** | MyEthereumのUIを複製した「Somora」アプリをTelegramリンクで配布。公式アプリストア外への配布でセキュリティ審査を回避。 |
| **投資チャンネルへの偽装** | 高利回り投資案内チャンネル/DMで信頼を構築した後、「専用投資プラットフォーム」のインストールを誘導。 |
| **Zoomミーティング詐称攻撃** | Telegramで知人を詐称 → Calendlyで予約 → 偽Zoom招待状を送付 → 「Zoom SDKアップデートスクリプト」の実行を誘導。1万行の空白にマルウェアを隠蔽。 |
| **AI深層偽造による身元詐称** | AI生成の偽の顔・身分証で投資家や著名人を詐称。ビデオ通話でも検出が困難なディープフェイクを活用。 |

### マルウェア機能一覧（NimDoorファミリー）

```
キーロガー         : 全キー入力を記録 → パスワード・シードフレーズを窃取
クリップボード監視 : コピーされたウォレットアドレスをハッカーのアドレスにリアルタイムで置き換え
ブラウザデータ     : 保存済みの認証情報・Cookie・セッションを窃取
ウォレット拡張機能 : MetaMaskを含む24以上のウォレット拡張データを窃取
リモート制御       : ウェブカメラ・マイクをリモート操作、ファイル転送・削除
Telegramデータ     : Telegramローカルデータ窃取スクリプト（tlgrm）を実行
持続性の維持       : LaunchAgent（macOS）に登録し、再起動後も自動実行
```

## 5. 開発者偽装侵入

2024〜2025年に最も顕著な戦術は、**ITエンジニア・開発者として偽装し、仮想通貨企業に正式採用される**方法である。Chainalysisのレポートによれば、2024年だけで12社以上の仮想通貨企業に北朝鮮の工作員が正規社員として侵入した。

> ⚠️ **境界セキュリティの無力化**  
> 攻撃者が合法的な内部アクセス権限を持つ場合、ファイアウォールやネットワーク境界セキュリティは効果がない。この戦術は業界全体に根本的な信頼危機をもたらす。

### 偽装採用攻撃のステップ

1. **偽の身元構築**——AI生成の写真、虚偽の職歴、幽霊会社の在職歴を組み合わせ、LinkedInとGitHubに一定期間の活動歴を積み上げる。
2. **リモート採用への応募**——ブロックチェーン開発者・スマートコントラクトエンジニアなど需要の高いポジションに応募。技術面接を通過できる実力を持つ。
3. **内部システムの偵察**——正規社員として勤務しながら、内部インフラ・セキュリティ構造・秘密鍵の管理方式を数ヶ月かけて把握。
4. **実行——内部から攻撃を支援**——認証情報の提供、セキュリティシステムの無効化、不正取引の承認などにより、外部のハッキングチームを内部から直接支援。

### GitHub偽装オープンソース攻撃

```
// 攻撃フロー（VSCode自動化機能の悪用）
Step 1: 正規のオープンソースプロジェクトに偽装した悪意ある
        リポジトリをGitHubにアップロード
Step 2: 開発者がリポジトリをクローンし、VSCodeでフォルダを開く
Step 3: .vscode/tasks.jsonが悪意あるスクリプトを自動実行
Step 4: マルウェア本体は.woff2や.svgに偽装してセキュリティスキャンを回避
Step 5: ブラウザ認証情報 + 24以上のウォレット拡張データを窃取
// 対象OS: Windows / macOS / Linux すべて対象
```

## 9. 防御チェックリスト

> 💡 **中核原則**: 北朝鮮の攻撃は技術的な脆弱性より人間の信頼と習慣を狙う。技術的なセキュリティ強化に加え、人・プロセス・サプライチェーンを中心とするセキュリティ文化が不可欠。

### 基本防御策

- [ ] **Telegram DM**: 知らない相手からの投資提案・ファイル・リンクには絶対に応じない。知人を詐称している場合は別のチャンネルで身元確認を。
- [ ] **ハードウェアウォレット**: 大口資産はコールドウォレットで保管。ホットウォレットには日常取引用の少額のみ。
- [ ] **クリップボードハイジャック**: 送金直前に必ず目視でウォレットアドレス全体を確認。クリップボード監視セキュリティツールを使用。
- [ ] **採用提案メール**: タスクファイルの実行要求はすぐに拒否。公式チャンネルで身元を別途確認。
- [ ] **オープンソースコード**: `.vscode/tasks.json`と`package.json`スクリプトを手動確認してから実行。VSCodeの自動実行を無効化。
- [ ] **サプライチェーン監査**: サードパーティのライブラリ・サービスを定期的に監査。外部開発者の採用時は徹底的な身元調査を実施。

### 二要素認証（2FA）——アカウント乗っ取りに対する最後の防衛線

> 🔐 **2FAなしのアカウントは鍵のない金庫も同然**  
> パスワードが盗まれても、2FAが有効なアカウントは攻撃者が2段階目の認証を突破できなければ侵入不可能。

| 方式 | セキュリティ強度 | 説明 |
|---|---|---|
| **ハードウェアセキュリティキー（FIDO2/U2F）** | 🟢 最強 | YubiKeyなどの物理キー。フィッシングサイトで入力してもドメイン不一致で認証自体が不可。取引所と高額資産アカウントに強く推奨。 |
| **TOTPアプリ（Google認証システム / Authy）** | 🟡 推奨 | 30秒ごとに更新される6桁OTP。SMSより安全でオフライン動作。バックアップコードは必ずオフラインで保管。 |
| **SMS 2FA** | 🟠 非推奨 | SIMスワッピング攻撃やSS7プロトコルの脆弱性で傍受可能。直ちにTOTPかハードウェアキーへ切り替えを。 |
| **メール 2FA** | 🔴 使用禁止 | メールアカウントはすでにフィッシングの主要ターゲット。メールが突破された瞬間に無力化。 |

**2FA優先順位**: 取引所アカウント > メールアカウント > Telegram/SNS > クラウドストレージ  
取引所のAPIキーを使用する場合は、**出金不可・読み取り専用**に権限を制限し、IPホワイトリストを併用することが必須。

### PC房（インターネットカフェ）・公共インターネット——仮想通貨へのアクセスは絶対禁止

> ⛔ **公共PCを使用した際に発生する4つの脅威**
> 1. **キーロガーのインストール有無が確認不可**——マルウェアやキーロガーがすでにインストールされていても目視では確認できない。
> 2. **ブラウザセッションの残留**——ログアウトしてもキャッシュやCookieに認証情報が残り、次のユーザーに露出する可能性がある。
> 3. **公共Wi-Fiでの中間者攻撃（MITM）**——カフェ・図書館・空港などのオープンネットワークでは、同じネットワーク上の攻撃者がトラフィックを傍受したり偽のアクセスポイントへ誘導したりすることができる。
> 4. **ショルダーサーフィン（Shoulder Surfing）**——シードフレーズやパスワードの入力が周囲の人や防犯カメラに露出する可能性がある。

**✅ 必ず守るべきルール：**
- [ ] 仮想通貨取引所・ウォレットへのアクセスは必ず本人所有のデバイス（スマートフォン・ノートPC）のみで行う
- [ ] やむを得ず公共PCを使用する場合、シードフレーズ・秘密鍵・パスワードは絶対に入力しない
- [ ] 公共Wi-Fiを使用する際は信頼できる有料VPNを必ず起動してから接続
- [ ] 取引所アプリは公式アプリストアからのみインストール——QRコードやリンクからのインストールは絶対禁止
- [ ] 画面共有中・配信中に2FA認証コードを絶対に露出しない（ストリーマー・YouTuberは特に注意）

**❌ 絶対にやってはいけないこと：**
- PC房使用後にブラウザ履歴を削除するだけで安心する（キーロガーが仕込まれていれば無意味）
- 個人のホットスポットなしにPC房の有線LANで個人ウォレット・取引所にアクセス
- 他人のデバイスで取引所にログイン（「ちょっと見るだけ」）

### 取引所・プロジェクト運営者専用チェックリスト

**✅ 必須実施項目：**
- [ ] ホットウォレットの資産比率を全体の10%以下に制限し、残りはコールドストレージに分離
- [ ] マルチシグウォレットの署名インフラ自体のセキュリティ監査——サードパーティサービスを含む（Bybitの教訓）
- [ ] すべての管理者アカウントにFIDO2ハードウェアキーベースの2FAを必須適用
- [ ] リモート採用時にディープフェイクのリスクを考慮した追加の身元確認手続きを導入
- [ ] 内部スタッフの権限アクセスログをリアルタイムで監視し、異常な行動を即時アラート
- [ ] 開発チームPCにエンドポイントセキュリティソリューションを展開し、外部ソフトウェアのインストールを制限

**❌ 厳重禁止事項：**
- 信頼できないソースのSDK・ライブラリを検証なしに本番環境へデプロイ
- 大規模な出金要求を自動化システムのみで処理（人間による検証ステップなし）
- 北朝鮮制裁関連のブロックチェーンアドレスDBとリアルタイムで照合せずに取引を承認
